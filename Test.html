Especificación del Proyecto - Cargue de archivos REDEBAN

Sistema de Gestión de Archivos y Consolidación de Datos en SQL Server

Descripción General del Proyecto

Este proyecto desarrolla un sistema que permite la gestión, descompresión, procesamiento y carga de archivos de datos hacia una base de datos SQL Server. La interfaz gráfica permite a los usuarios cargar archivos, verificar duplicados y personalizar la configuración a través de un archivo config.ini, optimizando la gestión de datos en un entorno de procesamiento intensivo.

Objetivos Específicos

  • Descompresión de archivos ZIP y verificación de duplicados.
  • Transformación y carga de datos en SQL Server.
  • Configuración flexible y personalizable mediante config.ini.
  • Registro de eventos y errores en un archivo de log.
  • Interfaz gráfica para una experiencia de usuario simplificada.

Especificaciones Técnicas

  • Lenguaje de Programación: Python 3
  • Framework de Interfaz Gráfica: customtkinter
  • Base de Datos: SQL Server
  • Archivo de Configuración: config.ini
  • Librerías Necesarias: pyodbc, pandas, tkinter, configparser, zipfile, logging, sys, os

Requerimientos del Sistema

  • Sistema Operativo: Windows
  • Python 3.6 o superior
  • Controlador ODBC para SQL Server
  • Permisos de escritura y acceso a SQL Server

Arquitectura del Proyecto

El sistema está compuesto por módulos independientes para la descompresión, filtrado y carga de archivos en SQL Server. Cada módulo está diseñado para cumplir con una funcionalidad específica y está integrado en una interfaz gráfica común.

Módulos y Funcionalidades

6.1 Módulo de Autenticación

Solicita credenciales y establece una conexión segura con SQL Server.

6.2 Módulo de Descompresión

Permite seleccionar una carpeta y descomprimir archivos ZIP, registrando los archivos procesados en una tabla de control.

6.3 Módulo de Filtrado y Procesamiento

Filtra y transforma los datos de acuerdo a criterios específicos definidos en el archivo config.ini antes de cargarlos en SQL Server.

6.4 Módulo de Configuración

Permite al usuario personalizar las rutas de archivos, criterios de filtrado y configuraciones de base de datos mediante config.ini.

Flujo del Sistema

  1. Inicio de sesión y autenticación.
  2. Descompresión de archivos ZIP.
  3. Filtrado y procesamiento de datos.
  4. Carga de datos en la base de datos.
  5. Modificación de configuración.

Archivos y Configuración

El archivo config.ini permite configurar la conexión a SQL Server, las rutas de archivos y criterios de filtrado.

[Database]
server = 127.0.0.1
database = REDEBAN
table_data = TablaDatos
table_control = TablaControl
table_control_descompresion = TablaControlDescompresion

[Paths]
source_path = ./input
destination_path = ./output
file_keyword = reporte

[Filters]
filtro_1_tipo_producto = ProductoX
filtro_1_privada = PrivadaA, PrivadaB
filtro_2_tipo_producto = ProductoY
filtro_2_tipo_operacion = OperacionZ

Pruebas y Validación

Las pruebas de este sistema se centran en verificar el correcto funcionamiento de cada módulo y la integridad de los datos en la base de datos.

  • Pruebas de autenticación y conexión.
  • Pruebas de descompresión y manejo de duplicados.
  • Pruebas de procesamiento y carga de datos con diferentes escenarios de filtros.
  • Pruebas de modificación y guardado de configuración.

Conclusiones

Este sistema permite una gestión automatizada de archivos de datos hacia SQL Server, con una interfaz intuitiva y un sistema de control de errores y duplicados robusto, facilitando su implementación en entornos de producción para el cliente REDEBAN.
